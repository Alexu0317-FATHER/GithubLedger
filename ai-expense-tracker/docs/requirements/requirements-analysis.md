## 项目概述

**项目名称**: GithubLedger - AI驱动的个人记账应用  
**核心价值主张**: 让记账从负担变成洞察，从被动记录变成主动决策助手

## 核心问题定义

基于用户描述的三大痛点：

### 1. 数据孤岛问题

- **现状**: 微信、支付宝、银行APP各自为政，缺乏统一视图
- **影响**: 用户无法获得全局消费认知，需要在不同软件间切换

### 2. 数据整合困难

- **现状**: 不同平台账单格式不统一，手动合并工作量巨大
- **影响**: 即使用户有心记账，技术门槛过高导致放弃

### 3. 个性化需求无法满足

- **现状**: 传统饼图和月度走势图无法解决实际生活场景问题
- **影响**: 用户无法获得真正有价值的消费洞察

## 具体个性化场景需求

### 场景3.1: 商品价格历史查询

- **典型场景**: 买猫粮时想知道之前的价格，是否涨价
- **现状痛点**: 需要去购物APP翻消费记录，费时费力
- **期望解决方案**:
  - 语音查询历史价格
  - 直观展示同类商品价格趋势
  - 自动识别商品涨跌幅

### 场景3.2: 消费控制与预警

- **典型场景**: Steam打折季疯狂购买游戏，或逛商场时冲动消费
- **现状痛点**: 难以控制自己的消费欲望，经常超支
- **期望解决方案**:
  - 基于历史数据掌握用户购物频率
  - 结合月收入自动评估超支风险
  - 在消费高峰期（如周末）提前发送个性化提醒

### 场景3.3: 极简记账体验

- **现状痛点**: 传统记账软件设置繁琐（输入金额+设置类别+添加备注）
- **用户放弃原因**: 操作复杂，难以长期坚持
- **期望体验**: 更轻松、智能的录入方式

## 目标用户定义

- **核心定位**: 数字化生活的普通个人用户（非企业，非亿万富豪）
- **收入范围**: 月收入3K-30K的广大工薪阶层
- **消费特点**: 多平台消费（微信、支付宝、银行卡、现金）
- **技术水平**: 基础数字化生活技能，但不愿意为记账投入过多学习成本
- **核心需求**: 简单、智能、统一的个人财务管理

**设计原则**:

- 不做过度细分：职场新人、家庭主妇、理财达人的核心需求本质相同
- 统一体验策略：AI根据用户行为自动适配，而非人为分类
- 普通人优先：专注服务千千万万的普通人，而非小众高端用户

## MVP功能与技术方案

### 核心功能模块

1. **账单导入合并模块**
   - ✅ **支持格式**: 仅Excel和CSV格式
   - 智能合并去重处理
   - 数据格式标准化

2. **智能分类模块**
   - ✅ **实现方式**: 基于规则逻辑，非AI驱动（MVP阶段）
   - 用户确认和修正机制
   - 分类规则持续优化

3. **个性化趋势分析模块**
   - ✅ **核心功能**: 基于用户历史数据的规则分析
   - 商品价格历史分析（参考场景3.1）
   - 消费行为模式识别（参考场景3.2）
   - **差异化价值**: 超越传统饼图和月度走势图

4. **前端界面模块**
   - 极简用户体验设计
   - 响应式Web界面
   - 直观的数据可视化

### 技术实现方案

**技术栈**:

- 前端: React + TypeScript + Chart.js/D3.js
- 存储: IndexedDB（大容量本地存储）
- 部署: GitHub Pages静态网站部署
- CI/CD: GitHub Actions自动部署

**平台策略**: Web应用优先 > 移动端 > 桌面应用

**数据安全**: MVP阶段本地存储 + 基础加密，正式版考虑云端同步

### 渐进式实现策略

- **MVP阶段**: 规则驱动的分析逻辑
- **迭代阶段**: 规则+AI混合模式
- **成熟阶段**: AI驱动的智能分析

### 数据获取方式

1. **手动导入账单** (MVP核心功能)
   - ✅ **支持格式**: CSV、Excel
   - 用户导入后确认分类
   - 定期提醒导入新账单

2. **API对接** (后续功能)
   - Google Sheets、Notion等平台集成
   - 考虑使用n8n等自动化平台实现

3. **手动记账** (辅助功能)
   - 提供快速录入入口
   - 智能建议分类和金额

## 核心差异化价值

本项目的差异化在于**个性化趋势分析**，超越传统记账软件的饼图和月度走势图：

- **针对具体商品的价格历史追踪**（场景3.1）
- **基于用户行为的消费模式识别**（场景3.2）
- **极简化的数据导入和分类体验**（场景3.3）
- **统一视图解决多平台数据孤岛问题**

**实现策略**: MVP阶段通过规则逻辑实现核心价值，避免AI复杂度的同时确保差异化体验。

5. **用户粘性**: 用户连续使用超过1个月的比例达到60%

### 长期成功指标

- **用户留存**: 月活跃用户保持增长
- **功能使用**: 个性化查询功能使用频率
- **商业价值**: 用户对高级功能的付费意愿

## 产品创意库 (Ideas Bank)

> **说明**: 记录项目的核心创意和未来拓展可能性，为后续迭代提供灵感来源

### 💡 已确认的核心创意

#### 个性化场景分析

- **商品价格趋势**: 基于历史消费数据，自动识别同类商品价格变化
- **消费习惯洞察**: 识别用户消费高峰期、冲动消费模式
- **超支风险预警**: 结合月收入和历史数据的智能提醒

### 🚀 未来拓展创意

#### 场景化个性提醒 (商业化潜力)

- **Steam打折季提醒**: "无人深空2.5折，请理性消费"
- **个性化促销信息**: 基于用户消费偏好的精准推送
- **商业模式**: 与行业合作，开放API，个性化信息植入
- **隐私考量**: 需要慎重处理用户数据，确保用户同意

#### 高级分析功能

- **用户群体对比**: "你的咖啡支出比同类用户高30%"
- **预测性分析**: "基于历史数据，你本月可能超支500元"
- **外部事件关联**: 整合促销、节日等外部事件数据

#### 技术增强

- **语音交互**: 语音查询历史价格和消费分析
- **OCR识别**: 拍照识别纸质账单
- **智能API对接**: 自动同步各平台消费数据

#### 商圈智能分析 (B端商业价值，低优先级)

- **核心理念**: 基于用户消费数据的商圈分析，为商家和投资者提供区域商业洞察
- **实现前提**: 需要足够的用户基数才能产生有价值的分析结果
- **商业潜力**: B端数据服务市场，避免C端隐私争议
- **注**: 此功能可能永远不会实现，仅作为长期商业想象记录

### 💰 商业模式探索

#### 收费策略思考

- **Token消耗模式**: 按AI调用次数收费（适合高级分析功能）
- **功能订阅模式**: 按高阶功能订阅（如详细报告、预测分析）
- **信息服务模式**: 个性化促销信息的商业合作分成

#### 实现方式权衡

- **AI方案**: 系统提示词 + AI分析（高精度，高成本）
- **规则方案**: 自研逻辑 + 数据分析（低成本，基于用户数据迭代）
- **混合方案**: 规则处理常见场景，AI处理复杂场景

## 关键决策记录

### ✅ AI功能实现方式

**决策**: MVP阶段采用规则驱动，避免AI API成本和复杂度  
**理由**: 基于用户真实数据建立规则更可靠，为后续AI优化奠定基础

### ✅ 账单导入格式

**决策**: MVP阶段仅支持Excel和CSV格式  
**理由**: 技术实现简单，用户覆盖率高，避免PDF解析复杂度
