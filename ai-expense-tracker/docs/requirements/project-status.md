## 项目概述

**项目名称**: GithubLedger - AI驱动的个人记账应用  
**项目目标**: 为普通人打造统一、智能、极简的个人财务管理工具  
**核心价值**: 让记账从负担变成洞察，从被动记录变成主动决策助手

## 关键决策记录

### 用户定位决策

- **最终确定**: 普通个人用户（非企业、非亿万富豪）
- **决策逻辑**: 避免用户群体过度细分，专注核心共同需求
- **用户特征**: 月收入3K-30K，多平台消费，技术门槛敏感

### 技术架构决策

- **平台优先级**: Web应用 > 移动端 > 桌面应用
- **存储策略**: MVP阶段本地存储，正式版考虑云端同步
- **AI集成**: ✅ **已确认 - MVP阶段不使用AI API，采用规则驱动**

### 产品策略决策

- **核心差异化**: 个性化趋势分析（超越传统饼图）
- **用户体验**: 极简导入流程，智能自动分类
- **实现方式**: 规则逻辑优于AI复杂度

### 隐私设计决策

- **设计原则**: 避免让用户感觉被"偷窥"
- **交互策略**: 用户主动触发 > 系统主动推送
- **数据透明**: 明确告知分析基于用户上传的账单数据

### 功能边界决策

- **账单格式**: ✅ **已确认 - 仅支持CSV/Excel格式**
- **分类逻辑**: ✅ **已确认 - 纯规则驱动，MVP阶段详细设计**
- **核心指标**: 10分钟用户体验，80%分类准确率

## 技术方案摘要

### 确定的技术栈

前端: React + TypeScript + Chart.js/D3.js
存储: IndexedDB (本地大容量存储)
分析方式: 规则驱动逻辑（MVP阶段不使用AI服务）
部署: GitHub Pages (静态网站)
CI/CD: GitHub Actions (自动部署)

### MVP功能模块

1. **数据导入模块**: 文件上传、格式解析、数据标准化（仅CSV/Excel）
2. **智能分类模块**: 基于规则的自动分类
3. **可视化模块**: 个性化仪表板、图表展示
4. **数据管理模块**: 本地存储、备份恢复

## 用户沟通记录

### 沟通风格和原则

- **分析框架**: 基于CRAFT原则（Core Logic, Risks, Alternatives, Feasibility, Test Cases）

## 继任者必知要点

### 🚫 绝对禁区（不可触碰）

1. **不要重新引入AI API**
   - MVP阶段已明确采用规则驱动
   - 这是经过深思熟虑的决策，不是技术限制

2. **不要设计"偷窥感"功能**
   - 避免主动推送个性化建议
   - 用户主动查询 > 系统主动推荐
   - 参考案例：用户反感淘宝的精准推荐

3. **不要扩大功能边界**
   - 仅支持CSV/Excel，绝不添加PDF/OCR
   - 10分钟用户体验是硬指标
   - 80%分类准确率是最低标准

### ✅ 核心设计原则

1. **规则驱动优于AI驱动**
   - 基于真实用户数据建立规则更可靠
   - 透明可控的逻辑比黑盒算法更适合MVP

2. **用户控制感优于系统智能**
   - 数据来源透明化
   - 用户主动触发分析
   - 明确告知分析基于上传的账单

3. **简单可行优于技术炫技**
   - 避免过度工程化
   - 专注核心价值验证
   - 技术服务于用户体验

### 🎯 MVP边界清晰定义

- **用户体验**: 新用户10分钟内完成完整流程
- **功能准确率**: 智能分类准确率80%以上
- **文件支持**: 仅CSV和Excel格式
- **存储方式**: 本地IndexedDB存储
- **分析方式**: 规则逻辑，非AI驱动
